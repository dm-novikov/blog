Когда данных больше чем твой RAM

Очень полезная возможность Pandas это возможность итерироваться по датафрейму, если вдруг он не влазит в память (такое часто бывает в реальной работе, особенно если вы возитесь с ETL).

data_iter = pd.read_csv('cool_dataset.csv', chunksize=N)

def data_prep(df, func):
    """ Отфильтруем данные и сложим в другой файлик
    """
    for chunk in df:
        tmp = func(chunk)
        tmp.to_csv('cool_dataset_prep.csv')

data_prep(data_iter, func_filter)
Сжимай свою дату

Pandas позволяет читать из запакованных данных, снижая тем самым I/O

data_frame = pd.read_csv('your_barby_size_dataset.csv', compression='gzip')
Используй enum как тип данных

Данные тип позволяет уменьшить объем данных, за счет того что будет хранить только числовые значения переменных, а текст будет лежать в хэш таблице (кстати это можно использовать когда считываешь данные из файла, параметр dtype, требуется передать {col:type})

data_frame['enum'].astype('category')

Важно, данный тип данных подходит только для столбцов с низкой вариабельностью. Иначе теряется смысл сего действия.

Снижай разрядность своего int/float

По умолчанию Pandas выделяет 64 бита для хранения ваших численных данных, но если вы уверены что такие цифры вам попросту не нужны, то измените тип на 32 или 16.

Используй временной индекс

Если ваши данные вам необходимо анализировать на временных промежутках, то вносите столбец под индекс, это сократит код, и позволит использовать встроенные функции для работы с временными рядами.

data_frame.index = pd.DatetimeIndex(list(data_frame['date']))

#Для примера разобьем наш дата фрейм на 10 частей

# Или построим простой график, без лишних телодвижений
data_frame.plot()
