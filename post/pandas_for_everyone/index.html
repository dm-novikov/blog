Хранение данных в Pandas

Jun 21, 2020
Когда данные больше чем твой RAM

Очень полезная фича Pandas это возможность итерироваться по датафрейму, если вдруг он не влазит в память (такое часто бывает в реальной работе, особенно если вы возитесь с ETL).

data_iter = pd.read_csv('cool_dataset.csv', chunksize=N)

def data_prep(df, func):
    """ Отфильтруем данные и сложим в другой файлик
    """
    for chunk in df:
        tmp = func(chunk)
        tmp.to_csv('cool_dataset_prep.csv')

data_prep(data_iter, func_filter)
Сжимай свою дату

Pandas позволяет читать из запакованных данных, снижая тем самым I/O

data_frame = pd.read_csv('your_barby_size_dataset.csv', compression='gzip')
Используй что то кроме objects

Сategory тип позволяет уменьшить объем данных, за счет того что будет хранить только числовые значения переменных, а текст будет лежать в словаре (кстати это можно использовать когда считываешь данные из файла при read_csv, параметр dtype, требуется передать {col:type})

data_frame['enum'].astype('category')

Важно, данный тип данных подходит только для столбцов с низкой вариабельностью. Иначе, вы не получите заметного улучшения.

Снижай разрядность своего int/float

По умолчанию Pandas выделяет 64 бита для хранения ваших численных данных, но если вы уверены что такие цифры вам попросту не нужны, то измените тип на 32.

Как посмотреть размер ваших столбоцов (Mb)
data_frame.memory_usage() / 1024

Небольшой пример того как простое преобразование уменьшило объем хранимых данных
